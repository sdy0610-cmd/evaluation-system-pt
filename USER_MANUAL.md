## 1. 시스템 개요

| 구분 | 내용 |
|------|------|
| 목적 | 창업중심대학 참여기업 서류·발표 평가 관리 |
| 사용자 | 관리자(admin) + 평가위원(evaluator) |
| 평가 단계 | 서류평가 → 발표평가 → 최종 결과 |
| 결과 구분 | 통과 / 예비 / 탈락 |
| 평가 방식 | 위원 5명 점수 중 최고·최저 제외, 나머지 3명 평균 |

### 화면 구성

```
로그인 화면
    ├── 관리자 (admin 역할)
    │     ├── 대시보드      — 전체 현황 한눈에 보기
    │     ├── 분과 관리     — 분과·전문기술분야 등록
    │     ├── 평가위원 관리 — 계정 생성 및 분과 배정
    │     ├── 기업 관리     — Excel 가져오기, 분과 배정
    │     ├── 점수 집계     — 점수 검토·수정·확정·결과 설정
    │     └── 결과 보고서   — Excel 내보내기
    │
    └── 평가위원 (evaluator 역할)
          ├── 담당 기업 목록
          └── 개별 기업 평가 (PDF 보기 + 점수 입력)
```

## 2. 전체 운영 흐름

```
[준비] 분과 등록 → 평가위원 등록 → 기업 Excel 가져오기 → PDF 업로드
         ↓
[서류평가] 평가위원 로그인 → 점수 입력
         ↓
[서류 집계] 관리자: 점수 검토 → 가점 입력 → 결과 설정(통과/예비/탈락) → 전체 확정
         ↓
[발표 진행] "발표평가로 진행" 버튼 → 통과/예비 기업이 발표 단계로 이동
         ↓
[발표평가] 평가위원 로그인 → 발표 기업 점수 입력
         ↓
[발표 집계] 관리자: 동일하게 점수 검토 → 최종 결과 설정
         ↓
[보고서] 결과 보고서 > Excel 내보내기
```

## 3. 관리자 매뉴얼

### 3-1. 초기 설정 순서

**처음 시스템을 쓸 때 반드시 이 순서대로 진행하세요.**

```
① 분과 등록  →  ② 평가위원 등록  →  ③ 기업 Excel 가져오기  →  ④ PDF 업로드
```

> 순서가 바뀌면 분과 배정이 안 되거나 PDF가 연결되지 않습니다.

---

### 3-2. 분과 관리

**경로**: 좌측 메뉴 → **분과 관리**

#### 분과 추가

1. **+ 분과 추가** 버튼 클릭
2. 입력 항목:
   - **구분 라벨**: `A`, `B`, `C`, `D` (또는 `정보통신1` 등 짧은 식별자)
   - **분과명**: `정보·통신`, `기계·소재 / 에너지·자원` 등 전체 명칭
   - **위원장명**: 해당 분과 위원장 이름
3. **저장** 클릭

#### 전문기술분야 추가

- 분과 카드 하단 **+ 전문기술분야 추가** 클릭
- 기업 Excel의 `전문기술분야` 컬럼 값과 **정확히 일치**해야 자동 배정됩니다

> 예) Excel에 `정보통신-SW` 라고 되어 있으면, 전문기술분야도 `정보통신-SW`로 등록

---

### 3-3. 평가위원 관리

**경로**: 좌측 메뉴 → **평가위원 관리**

#### 평가위원 개별 추가

1. **+ 위원 추가** 버튼 클릭
2. 입력 항목:

| 항목 | 설명 |
|------|------|
| 아이디 | 로그인 ID (영문+숫자, 공백 없음) |
| 이름 | 표시될 이름 |
| 비밀번호 | 초기 비밀번호 |
| 분과 | 소속 분과 선택 |
| 위원 순서 | 1~5 (점수 집계표에서 위원1~5 열에 매핑됨) |

> **위원 순서가 중요합니다.** 같은 분과 내 위원 5명에게 각각 1~5를 부여해야 합니다.

#### Excel로 일괄 등록

1. **Excel 가져오기** 버튼 클릭
2. 아래 형식의 Excel 파일 준비:

| 아이디 | 이름 | 비밀번호 | 분과라벨 | 위원순서 | 이메일 | 연락처 |
|--------|------|----------|----------|----------|--------|--------|
| eval01 | 홍길동 | pass123 | A | 1 | hong@email.com | 010-1234-5678 |
| eval02 | 김철수 | pass456 | A | 2 | kim@email.com | |

3. 파일 선택 후 **가져오기** 클릭

---

### 3-4. 기업 관리

**경로**: 좌측 메뉴 → **기업 관리**

#### Excel로 기업 가져오기

1. **Excel 가져오기** 버튼 클릭
2. 공문으로 받은 기업 명단 Excel 파일 선택
   - 시트명: `대상` (없으면 첫 번째 시트 사용)
   - 필수 컬럼: `과제번호`, `대표자명`, `과제명`, `전문기술분야`
3. **미리보기**에서 처음 20개 행 확인
4. **가져오기 확인** 클릭 → DB에 저장

#### 읽어오는 컬럼 목록

| Excel 컬럼명 | 저장 항목 |
|-------------|----------|
| 과제번호 | 기업 고유 ID (8자리 숫자) |
| 대표자명 | 대표자 |
| 과제명 | 사업 과제명 |
| 전문기술분야 | 기술 분야 |
| 평가기준 | 예비/초기/도약 |
| 모집공고 | 모집 유형 |
| 청/중장년 | 연령 구분 |
| 창업지역 | 지역 |
| 매출액 | 매출 (숫자) |
| 사원수 | 임직원 수 |
| 비고 | 특이사항 (레전드/제외/면제 자동 파싱) |

> **비고 컬럼 자동 파싱 규칙:**
> - `레전드` 포함 → 레전드 기업으로 표시
> - `제외：사유` 형식 → 심사 제외 처리
> - `서류평가면제(사유)` 형식 → 서류면제 처리

#### 분과 배정

기업 목록 각 행의 **분과** 드롭다운에서 직접 선택합니다.

- 분과 배정 전에는 평가위원이 해당 기업을 볼 수 없습니다
- 한 번에 여러 기업 배정: 테이블 상단 필터로 `분과 미배정` 선택 후 일괄 처리 가능

#### 기업 상세 편집

기업 행의 **편집** 버튼(또는 기업명 클릭) → 모달에서 모든 항목 수정 가능

### 3-5. 점수 집계

**경로**: 좌측 메뉴 → **점수 집계**

이 화면이 평가 운영의 핵심입니다.

#### 기본 사용법

1. **분과 선택** (드롭다운)
2. **서류평가 / 발표평가** 탭 선택
3. 테이블에서 각 기업의 위원1~5 점수, 평점, 가점, 최종점수, 순위 확인

#### 테이블 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| 순위 | 최종점수 기준 자동 산정 (제외·과락 기업 제외) |
| 과제번호 | 8자리 기업 ID |
| 위원1~5 | 각 평가위원 점수 (클릭 시 수정 가능) |
| 평점 | 최고·최저 제외 후 3명 평균 |
| 가점 | 클릭 시 가점 편집 모달 |
| 최종 | 평점 + 가점 합계 |
| 결과 | 통과 / 예비 / 탈락 드롭다운으로 직접 설정 |
| 확정 | ✅ = 전체 확정됨, 숫자 = 확정/전체 수 |

#### 점수 수정 (관리자 권한)

1. 수정할 점수 클릭 → 수정 모달 열림
2. **수정 점수** 입력 (0~100)
3. **수정 사유** 입력 (필수)
4. **저장** → 주황색으로 표시됨 (원점수와 구분)

> 확정된 점수는 수정할 수 없습니다.

#### 가점 입력

1. 가점 셀 클릭 → 가점 모달 열림
2. **가점1 / 가점2 / 가점3** 각각 점수 입력
3. 다른 곳 클릭 시 자동 저장
4. 합산값이 최종점수에 반영됨

#### 전체 확정

1. 결과 설정 완료 후 **전체 확정** 버튼 클릭
2. 확정 후에는 평가위원도 점수를 수정할 수 없음
3. 평가위원 화면에 **"확정됨"** 표시

#### 발표평가로 진행

서류평가 탭에서 결과 설정 완료 후:

1. **발표평가로 진행 →** 버튼 클릭
2. `통과` 또는 `예비`로 설정된 기업들이 자동으로 **발표** 단계로 이동
3. 이후 평가위원은 발표 기업 목록에서 발표평가 진행

## 4. 평가위원 매뉴얼

### 로그인

1. https://startup-selection-production.up.railway.app 접속
2. 관리자에게 받은 **아이디 / 비밀번호** 입력
3. **로그인** 클릭

### 기업 목록 화면

로그인하면 담당 분과의 기업 목록이 표시됩니다.

| UI 요소 | 설명 |
|---------|------|
| 검색창 | 기업명, 과제번호로 검색 |
| 단계 필터 | 전체 / 서류 / 발표 |
| 완료 필터 | 전체 / 완료 / 미완료 |
| 진행 바 | 상단에 완료 기업 수 / 전체 수 표시 |
| 카드 색상 | 회색=미평가, 파란 테두리=평가완료, 초록 배지=확정됨 |

### 평가 진행

1. 기업 카드 클릭 → 평가 화면으로 이동
2. **왼쪽**: 사업계획서 PDF 자동 표시
3. **오른쪽**: 점수 입력 패널

#### 점수 입력

- 슬라이더를 드래그하거나
- 숫자 입력란에 직접 숫자 입력 (0~100)

#### 점수 기준 참고

| 등급 | 점수 범위 | 의미 |
|------|----------|------|
| 탁월 | 90~100 | 매우 우수한 수준 |
| 우수 | 80~89 | 기준 이상 |
| 보통 | 70~79 | 평균 수준 |
| 미흡 | ~69 | 기준 미달 |

#### 평가의견 입력

- 하단 텍스트 영역에 종합 의견 작성 (선택사항)
- 예) *"기술 혁신성이 높으나 자금 조달 계획의 구체성이 부족합니다."*

#### 제출

- **평가 제출** 버튼 클릭 → 저장 완료
- 이후 수정하려면 다시 클릭 → **수정 저장** 버튼으로 변경됨
- 관리자가 **확정** 처리하면 이후 수정 불가

#### PDF가 보이지 않을 때

- 하단의 **"새 탭에서 PDF 열기"** 링크 클릭
- 그래도 안 보이면 관리자에게 PDF 파일 업로드 요청

## 5. PDF 파일 업로드 (Supabase Storage)

평가위원이 사업계획서를 볼 수 있으려면 관리자가 PDF를 업로드해야 합니다.

### 파일 명명 규칙

```
{과제번호}.pdf
예) 20318135.pdf
```

### 업로드 방법

1. [supabase.com](https://supabase.com) → 프로젝트 접속
2. 좌측 메뉴 → **Storage**
3. `startup-companies` 버킷 클릭
4. **Upload files** 버튼 → PDF 파일 선택 (다중 선택 가능)
5. 업로드 완료 후 시스템에서 자동 연결됨

> 버킷이 없으면: **New bucket** → 이름: `startup-companies` → **Public** 체크 → 생성

### 기업 관리에서 수동 연결

기업 상세 편집 → **PDF 경로** 항목에 `20318135.pdf` 입력 후 저장

## 6. 점수 계산 방식

### 평점 계산

```
평점 = (위원1 + 위원2 + 위원3 + 위원4 + 위원5 - 최고점 - 최저점) ÷ 3
```

**예시:**

| 위원1 | 위원2 | 위원3 | 위원4 | 위원5 |
|-------|-------|-------|-------|-------|
| 97 | 90 | 97 | 89 | 98 |

```
최고 = 98, 최저 = 89
평점 = (97 + 90 + 97 + 89 + 98 - 98 - 89) ÷ 3
     = 284 ÷ 3
     = 94.67
```

### 예외 처리

- 위원이 5명 미만인 경우: 제출된 점수의 단순 평균
- 과락(is_knockout): 순위에서 제외, 주황색 행으로 표시
- 제외 기업: 빨간 행으로 표시, 순위 미부여

### 최종점수

```
최종점수 = 평점 + 가점 합계 (가점1 + 가점2 + 가점3)
```

## 7. 자주 묻는 질문

**Q. 평가위원이 로그인이 안 됩니다.**
- 관리자: 평가위원 관리에서 해당 계정의 아이디/비밀번호 확인
- 아이디는 공백 없이 입력

**Q. 평가위원이 기업 목록을 볼 수 없습니다.**
- 기업에 분과가 배정되지 않은 경우입니다.
- 기업 관리 → 해당 기업 행의 분과 드롭다운에서 배정하세요.

**Q. PDF가 보이지 않습니다.**
- Storage에 `{과제번호}.pdf` 형식으로 파일이 업로드되었는지 확인
- Supabase Storage 버킷이 **Public**으로 설정되어 있는지 확인

**Q. 점수를 확정했는데 취소하고 싶습니다.**
- Supabase Dashboard → SQL Editor에서 직접 처리:
  ```sql
  UPDATE startup_evaluations
  SET is_confirmed = false, confirmed_at = null
  WHERE company_id = '과제번호' AND evaluation_type = '서류';
  ```

**Q. 기업을 잘못 가져왔습니다. 삭제하고 싶습니다.**
- Supabase Dashboard → SQL Editor:
