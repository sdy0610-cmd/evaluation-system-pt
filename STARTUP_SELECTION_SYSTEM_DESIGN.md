> 작성일: 2026-03-18

## 1. 현행 채용평가 시스템과의 비교

| 항목 | 채용평가 시스템 | 선발평가 시스템 |
|------|----------------|----------------|
| 평가 대상 수 | 수십 명 | ~2,000개 기업 |
| 평가 단계 | 1회 | 서류평가 → 발표평가 (2단계) |
| 평가위원 수 | ~5명 | 분과당 5명 × 최대 분과 수 |
| 분야/분과 | 없음 | 분야별 분과 구성 (최대 분과 15개) |
| 지원자 관리 | Supabase 직접 입력 | Excel 업로드 (접수번호 기반) |
| 평가위원 관리 | Supabase 직접 입력 | Excel 업로드 또는 직접 입력 |
| 결과 보고서 | 단일 표 | 분야별·분과별 분리 보고서 |

## 2. 핵심 데이터 구조 설계

### 데이터 계층 구조

> **실제 운영 데이터 기반** — `발표집계_창중.xlsx` 분석 결과 반영

```
연도(Year)
 └─ 분과(Division)  예: A(정보통신), B(기계ㆍ소재/에너지ㆍ자원), C(공예ㆍ디자인/화공ㆍ섬유)...
     ├─ 평가위원(Evaluator) × 5명 + 위원장(Chair) × 1명
     └─ 기업(Company) × N개  [전문기술분야 복수 포함 가능]
```

> **중요**: 분과는 여러 전문기술분야를 묶을 수 있음.
> 예: B분과 = "기계ㆍ소재 / 에너지ㆍ자원" (두 분야가 같은 분과)
> 정보통신은 기업 수가 많아 정보통신1 / 정보통신2 / 정보통신3 으로 분리.

### 실제 분과 구성 예시 (2025년도)

| 분과 | 분과명 | 포함 전문기술분야 |
|------|--------|------------------|
| A | 정보·통신 | 정보통신1 / 정보통신2 / 정보통신3 |
| B | 기계·소재 / 에너지·자원 | 기계ㆍ소재, 에너지ㆍ자원 |
| C | 공예·디자인 / 화공·섬유 | 공예ㆍ디자인, 화공ㆍ섬유 |
| D | 바이오·의료·생명 / 전기·전자 | 바이오ㆍ의료ㆍ생명, 전기ㆍ전자 |

### Supabase 테이블 설계

```sql
-- 분과 (매년 재구성 가능, 여러 전문기술분야 포함 가능)
CREATE TABLE divisions (
  id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  year            INTEGER NOT NULL,
  division_label  TEXT NOT NULL,     -- 'A', 'B', 'C', 'D' 또는 '정보통신1' 등
  division_name   TEXT NOT NULL,     -- '정보·통신', '기계·소재 / 에너지·자원' 등
  chair_name      TEXT,              -- 위원장 성명
  UNIQUE(year, division_label)
);

-- 전문기술분야 (분과에 속함, 1개 분과에 복수 분야 가능)
CREATE TABLE tech_fields (
  id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  year            INTEGER NOT NULL,
  division_id     UUID REFERENCES divisions(id),
  field_name      TEXT NOT NULL,     -- '기계ㆍ소재', '에너지ㆍ자원', '정보ㆍ통신' 등
  UNIQUE(year, field_name)
);

-- 평가위원
CREATE TABLE evaluators (
  id          TEXT PRIMARY KEY,       -- 로그인 ID (예: tdtd042)
  name        TEXT NOT NULL,
  password    TEXT NOT NULL,
  role        TEXT DEFAULT 'evaluator', -- 'evaluator' | 'admin'
  division_id UUID REFERENCES divisions(id),
  evaluator_order INTEGER,            -- 분과 내 위원 순서 (위원1~5)
  email       TEXT,
  phone       TEXT
);

-- 지원기업
CREATE TABLE companies (
  project_no       TEXT PRIMARY KEY,   -- 과제번호 (예: 20318135) — 시스템 로그인 ID와 동일
  company_name     TEXT NOT NULL,
  representative   TEXT,              -- 대표자명
  birth_year       TEXT,              -- 생년월일 앞 4자리
  gender           TEXT,              -- '남' | '여'
  age_group        TEXT,              -- '청년' | '중장년'
  startup_stage    TEXT,              -- '예비' | '초기' | '도약'
  recruit_type     TEXT,              -- '①권역 내 일반형' | '②지역주력산업 연계형' 등
  region           TEXT,              -- 창업지역 (서울, 경기, 인천...)
  support_field    TEXT,              -- 지원분야 (제조, 지식서비스...)
  tech_field       TEXT,              -- 전문기술분야 (정보ㆍ통신, 기계ㆍ소재 등)
  ict_sub_field    TEXT,              -- 정보통신 세부분야 (SW, 제조 등), 해당 없으면 NULL
  division_id      UUID REFERENCES divisions(id),
  revenue          BIGINT,            -- 매출액 (원)
  employees        INTEGER,           -- 사원수
  contact_phone    TEXT,
  contact_mobile   TEXT,
  email            TEXT,
  login_id         TEXT,              -- 시스템 로그인 ID (과제번호와 별도 운용 가능)
  address          TEXT,
  project_title    TEXT NOT NULL,     -- 과제명
  is_legend        BOOLEAN DEFAULT FALSE,  -- 레전드 기업 여부
  is_excluded      BOOLEAN DEFAULT FALSE,  -- 평가 제외 여부
  exclusion_reason TEXT,              -- 제외 사유 (초기창업패키지, 레전드 등)
  is_doc_exempt    BOOLEAN DEFAULT FALSE,  -- 서류평가 면제 여부
  doc_exempt_reason TEXT,             -- 면제 사유 (투자사명 등, 예: '퓨처플레이')
  employment_target INTEGER,          -- 고용목표
  revenue_target   BIGINT,            -- 매출목표 (원)
  investment_target BIGINT,           -- 투자목표 (원)
  requirement_check TEXT,             -- 요건검토 결과
  eval_date        TEXT,              -- 평가일자
  notes            TEXT,              -- 비고
  stage            TEXT DEFAULT '서류',  -- '서류' | '발표' | '통과' | '예비' | '탈락'
  file_path        TEXT               -- Supabase Storage 경로 (과제번호.pdf)
);

-- 가점 (기업별 다항목 가점)
CREATE TABLE bonus_points (
  id         BIGSERIAL PRIMARY KEY,
  company_id TEXT REFERENCES companies(project_no),
  year       INTEGER NOT NULL,
  bonus_type TEXT NOT NULL,   -- 가점1, 가점2, 가점3 구분
  reason     TEXT,            -- 가점 사유
  points     NUMERIC(5,2) DEFAULT 0
);

-- 평가 결과 (평가위원 1인 × 기업 1개 × 평가단계)
CREATE TABLE evaluations (
  id                   BIGSERIAL PRIMARY KEY,
  company_id           TEXT REFERENCES companies(project_no),
  evaluator_id         TEXT REFERENCES evaluators(id),
  evaluation_type      TEXT NOT NULL,       -- '서류' | '발표'
  score                NUMERIC(5,2),        -- 해당 위원의 점수 (100점 만점)
  comment              TEXT,               -- 평가의견 (자유 서술형)
  is_knockout          BOOLEAN DEFAULT FALSE, -- 과락 여부
  adjusted_score       NUMERIC(5,2),        -- 관리자 수정 점수 (수정 없으면 NULL)
  adjusted_by          TEXT,                -- 수정한 관리자 ID
  adjusted_at          TIMESTAMPTZ,
  adjustment_reason    TEXT,
  is_confirmed         BOOLEAN DEFAULT FALSE,
  confirmed_at         TIMESTAMPTZ,
  submitted_at         TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(company_id, evaluator_id, evaluation_type)
);

-- 분과별 집계 결과 (캐시 테이블 — 조회 성능용)
CREATE TABLE division_scores (
  id             BIGSERIAL PRIMARY KEY,
  company_id     TEXT REFERENCES companies(project_no),
  evaluation_type TEXT NOT NULL,            -- '서류' | '발표'
  score_v1       NUMERIC(5,2),              -- 위원1 점수
  score_v2       NUMERIC(5,2),              -- 위원2 점수
  score_v3       NUMERIC(5,2),              -- 위원3 점수
  score_v4       NUMERIC(5,2),              -- 위원4 점수
  score_v5       NUMERIC(5,2),              -- 위원5 점수
  avg_score      NUMERIC(5,2),              -- 평점 (최고·최저 제외 후 3점 평균)
  final_score    NUMERIC(5,2),              -- 최종 점수 (평점 + 가점)
  rank_in_division INTEGER,                 -- 분과 내 순위
  result         TEXT,                      -- '통과' | '예비' | '탈락'
  is_confirmed   BOOLEAN DEFAULT FALSE,
  updated_at     TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(company_id, evaluation_type)
);
```

## 3. 평가 기준 (채점표)

### 점수 계산 공식 — 핵심

> **`발표집계_창중.xlsx` 집계 시트에서 확인된 실제 공식**

```
평점 = (위원1 + 위원2 + 위원3 + 위원4 + 위원5 - 최고점 - 최저점) / 3
최종점수 = 평점 + 가점합계
```

- 5명 위원 중 **최고점 1개, 최저점 1개를 제외**하고 나머지 3점의 평균
- 동점자 최고·최저 처리: 최고점 중복 시 1개만 제외, 최저점 동일 적용
- 소수점 이하 유지 (예: 94.67)

### 평가 항목 (관리자 설정 가능)

서류평가와 발표평가는 **항목별 배점 없이 100점 만점 단일 점수** 입력 방식 사용 가능.
항목별 세분화 입력 방식은 관리자 설정으로 전환 가능.

> 항목별 입력 방식 채택 시 예시 (관리자 설정 페이지에서 매년 수정 가능):

**서류평가 (100점 만점 예시)**

| 항목 | 배점 |
|------|------|
| 사업계획 혁신성 | 20점 |
| 시장성 및 성장가능성 | 20점 |
| 창업팀 역량 | 20점 |
| 사회·경제적 가치 | 20점 |
| 사업 실현가능성 | 20점 |

**발표평가 (100점 만점 예시)**

| 항목 | 배점 |
|------|------|
| 발표 내용 충실성 | 20점 |
| 사업모델 차별성 | 20점 |
| 시장 분석력 | 20점 |
| 팀 실행력 | 20점 |
| Q&A 대응 능력 | 20점 |

### 가점 제도

실제 Excel에서 `가점1`, `가점2`, `가점3` 항목이 별도 관리됨.

| 가점 유형 | 설명 |
|-----------|------|
| 가점1 | (사업별 기준 상이 — 관리자 설정) |
| 가점2 | (사업별 기준 상이) |
| 가점3 | (사업별 기준 상이) |
| 가점 합계 | 최종 점수에 합산 |

### 과락 기준

- `과락` 필드: 특정 점수 이하 또는 특정 항목 기준 미달 시 과락 처리
- 과락 기업은 순위 산정에서 제외, 결과 = '탈락' 자동 설정
- 과락 기준 점수는 관리자가 설정

### 서류평가 면제

투자사 추천 등 일정 요건 충족 시 서류평가 생략 가능.
- `is_doc_exempt = TRUE` 설정 + 면제 사유 기록 (예: '퓨처플레이', '블루버스인베스트먼트')
- 면제 기업은 서류평가 단계를 건너뛰고 발표평가로 직행

### 결과 상태 (3단계)

| 상태 | 의미 |
|------|------|
| 통과 | 발표평가 진출 확정 |
| 예비 | 대기 (추가 선발 가능) |
| 탈락 | 미선발 |

> 기존 설계의 '선발'/'미선발' 대신 실제 Excel 운영 기준인 **통과/예비/탈락** 3단계 사용

### 평가의견 (자유 서술형)

- 위원별로 기업에 대한 서술형 의견 입력 (100~500자 내외)
- 5명 위원의 의견이 각각 저장되며, 채점표/보고서 출력 시 번호(①②③④⑤)와 함께 표시
- Excel에서는 `_x000D_` 구분자로 5명 의견 연결 저장 → 시스템에서는 별도 행으로 관리

## 3-1. 점수 확정 및 채점표 출력 프로세스

### 점수 흐름 (원본 → 수정 → 확정)

```
평가위원 점수 입력 (scores / total)
          ↓
관리자 검토 — 필요 시 항목별 점수 수정
  · adjusted_scores / adjusted_total 에 저장
  · 수정 사유(adjustment_reason) 기록
  · 수정 이력 유지 (원본 scores 는 절대 덮어쓰지 않음)
          ↓
관리자 최종 확정 (is_confirmed = TRUE)
  · 확정 후에는 평가위원도, 관리자도 수정 불가
  · 확정 취소는 관리자 전용 별도 버튼으로만 가능
          ↓
채점표 출력 — 확정된 점수 반영
```

### 출력 채점표 구성 (평가위원별 1장)

채점표는 **평가위원 1인 × 담당 기업 전체**를 한 장에 출력합니다.
서류평가 채점표와 발표평가 채점표를 각각 별도로 출력합니다.

**채점표 상단 헤더**
- 사업명 (예: 2024년도 창업중심대학 참여기업 선발)
- 분야명 / 분과명
- 평가 단계 (서류평가 / 발표평가)
- 평가위원 성명

**채점표 본문 (테이블)**

| 접수번호 | 기업명 | 항목1 | 항목2 | 항목3 | 항목4 | 항목5 | 합계 | 비고 |
|---------|--------|-------|-------|-------|-------|-------|------|------|
| 2024-ICT-001 | 테크스타트업 | 18 | 16 | 17 | 15 | 19 | **85** | |

- 수정된 점수가 있는 경우: 수정 전 점수를 ~~취소선~~으로 표시하고 수정 후 점수를 굵게 표기
- 비고 컬럼: 관리자 수정 사유 간략 기재

**채점표 하단 서명란**

```
위 평가 결과가 사실임을 확인합니다.

평가일:  년   월   일

평가위원:                  (서명)
```

## 4. 파일 네이밍 규칙 (PDF)

Supabase Storage 버킷: `companies`

```
{과제번호}.pdf
예: 20318135.pdf
    20326027.pdf
    20321612.pdf
```

Excel 업로드 시 `과제번호` 컬럼과 Storage 파일명이 자동 매칭됩니다.
필요 시 `{과제번호}_{기업명}.pdf` 형식도 파싱 지원 (언더스코어 기준 분리).

> **변경**: 기존 `접수번호` (2024-ICT-001 형식) → 실제 운영 기준인 `과제번호` (8자리 숫자) 로 변경

## 5. Excel 업로드 형식

> 실제 `발표집계_창중.xlsx`의 **대상** 시트 컬럼 구조 기반

### 지원기업 업로드 (`기업목록.xlsx`) — 대상 시트 기준

| 컬럼명 | 필수 | 설명 | 예시 |
|--------|------|------|------|
| 과제번호 | ✅ | 8자리 고유번호 (PK) | 20318135 |
| 대표자명 | ✅ | | 이정현 |
| 평가기준 | | 예비/초기/도약 | 초기 |
| 모집공고 | | ①권역 내 일반형 등 | ①권역 내 일반형 |

## 7. 2단계 평가 플로우

```
① 지원기업 Excel 업로드 (대상 시트 형식) + PDF Supabase 업로드 (과제번호.pdf)
          ↓
② 분과 구성 (A/B/C/D 또는 정보통신1/2/3 등) + 위원장·평가위원 배정
   · 레전드 기업 표시, 제외 기업 처리, 서류평가 면제 기업 설정
   · 가점 항목 입력 (가점1/2/3)
          ↓
③ [서류평가] 평가위원 5명 × 담당 기업 채점 + 평가의견 입력
   · 서류면제 기업은 이 단계 스킵
          ↓
④ 관리자: 집계 시트 형식으로 점수 검토
   (위원1~5 점수, 평점=최고·최저 제외 3점 평균, 과락 처리)
   · 필요 시 점수 수정 (원본 보존, 수정 사유 기록)
          ↓
⑤ 관리자: 서류평가 점수 최종 확정 (확정 후 수정 잠금)
          ↓
⑥ [서류평가 채점표 출력] 평가위원별 1장씩 인쇄 → 평가위원 서명 수령
          ↓
⑦ 관리자: 확정 점수 + 가점 기준 순위 산정 → 통과/예비/탈락 결정
   · 종합 현황표 생성 (분과별 대상/면제/통과/예비/탈락 수)
          ↓
⑧ [발표평가] 통과 기업 발표 → 평가위원 채점 + 평가의견 입력
          ↓
⑨ 관리자: 점수 검토·수정·확정 (서류평가와 동일 절차)
          ↓
⑩ [발표평가 채점표 출력] 평가위원별 1장씩 인쇄 → 평가위원 서명 수령
          ↓
⑪ 최종 결과 보고서 생성
   · 집계 시트 (과제번호, 대표자명, 분과, 순위, 위원1~5, 평점, 결과, 의견)
   · 종합 시트 (분과별 통계)
   · 보고 시트 (순위, 분과, 과제번호, 가점, 점수, 결과 등)
   · 통과 시트 (통과 기업 상세 정보)
   · Excel 내보내기
```

| 9단계 | 평가위원 - 발표평가 화면 | 5단계와 동일 구조 |
| 10단계 | 결과 보고서 + Excel 내보내기 | 집계/종합/보고/통과 시트 형식 그대로 내보내기 |
